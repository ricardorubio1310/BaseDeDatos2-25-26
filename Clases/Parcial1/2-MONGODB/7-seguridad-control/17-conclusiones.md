# Conclusión

A lo largo de esta clase hemos abordado una dimensión fundamental que complementa todos los temas técnicos estudiados anteriormente: la operación segura y confiable de una base de datos MongoDB.

Primero analizamos cómo proteger el acceso mediante autenticación, roles y validación estructural de documentos. Estas herramientas permiten controlar quién accede al sistema y garantizar que los datos almacenados mantengan una estructura coherente.

Posteriormente estudiamos mecanismos de resiliencia operativa, como las estrategias de respaldo y recuperación. Estas prácticas permiten enfrentar incidentes inevitables en sistemas reales, desde errores humanos hasta fallos de infraestructura.

Finalmente introdujimos conceptos de observabilidad y auditoría, que permiten comprender el comportamiento del sistema y mantener trazabilidad sobre las operaciones realizadas.

Con estos conocimientos se completa prácticamente el ciclo completo de diseño y operación de bases de datos documentales en MongoDB:

* modelado de documentos
* optimización de consultas
* diseño de índices
* control de acceso
* validación estructural
* respaldo y recuperación
* monitoreo y auditoría
